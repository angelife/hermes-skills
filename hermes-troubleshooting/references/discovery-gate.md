# Discovery Gate（新事物评估强制检查点）

> 路径：`references/discovery-gate.md`
> 归属：`hermes-troubleshooting` skill
> 分类：方法论
> 关联：`SKILL.md` → 主引用（待 SKILL.md 瘦身后添加指向行）

## 触发条件

对话中任何一方（用户或 Hermes 自己）提到此前系统里不存在的工具/项目/模型/框架/服务，且暗示"可以试试"或"要不要用"。

## 触发后的强制表格

Hermes 必须在给出任何"建议安装/建议尝试"类结论之前，先按以下表格输出评估（不允许跳过任何一格，不知道就写"未知，需查证"）：

| 项目 | 内容 |
|---|---|
| 是什么 | 一句话，不超过30字 |
| 解决什么问题 | 具体到：现在因为没有它，卡在哪一步 |
| 跟现有系统关系 | 是替代（哪个）/补充（哪个）/无关 |
| 部署成本 | 预估：分钟级/小时级/天级，加一句理由 |
| 维护成本 | 一次性 还是 长期需要盯（更新/密钥轮换/兼容性） |
| 值不值 | 结论标签（见下） |

## 结论标签

只能是以下四个之一，不允许输出"建议直接安装"这类操作性结论：

- **值得测试** — 部署成本低，先花20分钟验证可行性
- **值得研究** — 收益不确定，先看文档/issue，不动手装
- **值得记录** — 现在用不上，但未来可能有用，存进候选清单
- **忽略** — 跟现有系统冲突/收益不明显/维护成本过高

## 例外

如果用户明确说"我知道这是什么，直接帮我装"，跳过此检查点，直接执行——这条规则是防止冲动决策，不是阻拦已经想清楚的决定。
