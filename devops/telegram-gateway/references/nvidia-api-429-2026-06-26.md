# Session: NVIDIA API 429 vs Telegram 429 — Bot Silent / No Reply

**Date:** 2026-06-26
**Symptom:** Bot stops replying. `gateway.log` shows polling healthy (Connected, periodic refresh working, inbound messages received). Bot cannot send responses.

## Root Cause

Upstream **model provider (NVIDIA) rate limit** — not Telegram.

```
gateway.error.log:
WARNING agent.conversation_loop: API call failed (attempt 1/3) error_type=RateLimitError
  provider=nvidia base_url=https://integrate.api.nvidia.com/v1 model=minimaxai/minimax-m2.7
  summary=HTTP 429: Error code: 429 - {'status': 429, 'title': 'Too Many Requests'}
```

This error does NOT appear in `gateway.log`. `gateway.log` only shows:
- Polling OK
- Inbound message received
- Nothing about the model call failure

## Distinction

| Source | Log File | Appears in gateway.log? |
|--------|----------|------------------------|
| Telegram Bot API 429 | gateway.log | Yes |
| Upstream model provider 429 | gateway.error.log | No |

## Fix

Restart gateway to refresh connection pool and retry:

```bash
launchctl unload ~/Library/LaunchAgents/ai.hermes.gateway.plist
sleep 3
launchctl load ~/Library/LaunchAgents/ai.hermes.gateway.plist
```

If persistent after restart, wait for provider rate limit window (typically minutes to ~1 hour for NVIDIA integrate.api.nvidia.com).

## Diagnostic Command

```bash
# Check gateway.error.log for upstream model 429s
tail -50 /Users/macos/.hermes/logs/gateway.error.log | grep "RateLimitError\|429\|API call failed"

# Check gateway.log for Telegram-side issues
tail -50 /Users/macos/.hermes/logs/gateway.log | grep "429\|Pool timeout\|Connected"
```
