# iOS → Android WeChat 数据迁移

## 场景

iPhone 上的微信账号积累了完整的聊天记录（多于 Android 端），需要迁移到 Android 以便用 root + ADB 拉取解密。

## 限制

- 同一微信账号**不能同时登录两个手机端**
- 迁移完成后 Android 登录 → iPhone 被踢下线
- 跨平台迁移不迁移设置/联系人/登录态，只迁移聊天记录和附件

## 迁移步骤

1. **两手机连同一 WiFi**
2. **Android 不要先登录**（不能同时在线）
3. **iPhone** → 我 → 设置 → 通用 → 聊天记录迁移与备份 → **迁移到另一台手机**
4. 选择要迁移的聊天（按会话选择或全选）→ 生成二维码
5. **Android 登录同微信账号** → 扫这个码
6. 等待传输完成（数据量越大越慢，同 WiFi 下约 1-5MB/s）

## 迁移后注意事项

- **ADB WiFi 调试会失效**：迁移后 Android 无线 ADB 调试模式（`adb tcpip 5555`）会被复位。需要插 USB 重新启用：
  ```bash
  adb usb && adb tcpip 5555
  # 拔 USB
  adb connect <new_ip>:5555
  ```
- **数据库增大**：迁移后 EnMicroMsg.db 会显著增大（从数 MB 到数百 MB），解密步骤不变（同一个 key “0273023” 仍有效）。
- **迁移中途中断可续传**：iPhone 微信会提示断点续传。

## 迁移后验证（2026-07-12 实测数据点）

- 同账号，Android 端原数据：1,404 条消息，31MB 加密库
- iPhone 端迁移后 Android 端：**10,710 条消息，51MB 加密库**
- 迁移前只有 1 个群聊，迁移后出现 **4 个群聊**（主群从 9 条消息涨到 8,833 条）
- 迁移后 key 不变（`0273023`），解密参数不变（`cipher_compatibility=1`）
- 迁移后无线 ADB 调试会失效，需要插 USB 重新启用：`adb usb && adb tcpip 5555`
