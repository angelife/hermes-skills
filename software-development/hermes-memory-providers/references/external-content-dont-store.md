# 为什么外部内容不进 Hindsight

## 背景
2026-07-13 从微信群 82 篇公众号文章（type=49, mp.weixin.qq.com）直接存入 Hindsight。用户问"把文章存进记忆的目的是啥呢"后纠正。

## 原因

| 问题 | 说明 |
|------|------|
| 语义不匹配 | Hindsight 的 observation/fact 模型是为"个人经验"设计的，不是为"外部文章索引" |
| Token 浪费 | 82 篇文章被 Hindsight 异步处理，调用了 LLM 抽取实体/关系，产生约 13 个节点但几乎无实际使用价值 |
| 检索无意义 | 永远不会用语义搜索去搜"昨天群里发了哪篇公众号"——直接在微信里翻更快 |
| 污染记忆 | 外部链接把真实的个人决策/观察淹没在信息流中 |

## 正确流程

1. 拉群文章 → 扫标题，挑出跟项目相关的
2. 问用户"要不要看这篇？"
3. 读后有收获 → 把个人洞察/决策理由提炼为 observation 存入 Hindsight
4. 文章链接本身 → 不进记忆，最多推荐给用户时贴一下

## 这个错误对应的用户原话

- "怎么变成memory server了 不是hindsight么"
- "文章就不该放进记忆"
