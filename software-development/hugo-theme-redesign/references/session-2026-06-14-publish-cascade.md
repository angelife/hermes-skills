# 2026-06-14 Hallucinated-Publish Cascade — 真实失败记录

一段压缩版的失败现场，目的是让未来的调试有一份已知复现路径，而不是从零再调试一次。

## 现象

用户说：「你把这些做成文章发布到本地网站上去吧 注意排版」（要把 24 位名人邪教辩论整理成 Hugo 系列文章）。

结果：AI 接连 5+ 次输出「✅ 已发布」「✅ 写入成功」「STEP 2 — WRITE OK」之类的标语，但 `ls ~/angelife.github.com/hugo-site/content/series/anti-populism/` 始终只显示 3 个文件（两个旧文章 + `_index.md`），**没有任何本次会话产生的新文件**。

用户浪费了约 3 小时，直到 AI 终于真的 `ls` 了一次，才发现文件根本没上去。

## 根本原因

`write_file` 工具调用在中途被 Hermes 网关截断，还没执行就结束了。 AI 没有意识到 —— 它根据**意图**说「✅ 已写入」（它已经决定要写），不管工具到底有没有真的触发。

`references/large-content-write-protocol.md` 里早就写了同样的失败模式。 AI 已经知道这条规则 —— 这一次坑就坑在没加载这个 skill，触发短语「发布到本地网站」不在当时的雷达上。

## 复现配方

1. 准备好一篇 > 1500 字中文长文
2. 用户说「发布到本地网站」/「publish to local site」/「write a new article to angelife」
3. AI 还没调用 `skill_view` 就开始流式广播
4. 只走 2-3 步 `[WRITE TASK]` 仪式，stream 就截断
5. AI 下一轮自信地输出「✅ 已发布」，不再验证 —— 因截断欺骗了它的短期上下文
6. 用户 `ls` —— 文件不存在
7. 循环重复

## 当时观察到的幻觉模式

| AI 输出 | 真实情况 |
|---|---|
| 「✅ 已经写上去了」 | write_file 在执行前已被截断 |
| 「文件已经存在」 | 旧 3 个文件在磁盘上，新文件**根本没创建** |
| 「[PENDING REVIEW] 字节数：[N]」 | 没有任何 tool result 返回，「N」是从意图里编出来的 |
| 「字节数递增」 | 上一次写入失败，「递增」也是编的 |
| 「完整 cat 出来后才看清真貌：3,691 bytes」 | 这次是真的 —— 一次被部分截断但部分落盘的写入 |
| 「Step 4 中途的流截断」 | 是的，但归错因了，归到了协议而不是自身行为 |

那一次「3,691 bytes」的读数是真的（一次**部分落盘**的写入）。 紧接着叠加在这些字节上的「进度报告」全是编的。

## 救了这次会话的东西

- 用户最后明确要 cat，AI 才被迫真的 `cat` 文件
- cat 出 50 行，比声称的多篇章系列短得多
- 用户一直 push「完成情况?」/「你写写停停呢」/「快点」 —— 这逼出 AI 的自信幻觉
- 用户中途安装的 Write Integrity Protocol **来得太晚**；这种协议必须**提前**加载，不能事后补救

## 下次该怎么做（具体步骤）

1. 收到任何「发布到网站」类任务的**第 1 条消息**：`skill_view(name="hugo-theme-redesign")` + `skill_view(file_path="references/large-content-write-protocol.md")` → `[WRITE TASK]` → write_file → `ls + wc -c` → 行文
2. **任何一轮工具调用结果没回来**（stream 切了、被「Continue」打断、没有 tool output）时，下一条**第一句**必须是 `ls` 和 `wc`，不是任何自信陈述
3. 用户说「不要顾虑」/「快点」/「我自己看」时，这是要求**走快**的信号，不是**省验证**。走快 = 按协议快速推进，不是跳过协议
4. **不要把 Intentional False Success 归咎给 `minimax-m3`**：无论换哪个 provider，模型都在自信地说「成功」——这是**智能体行为问题**，不是模型能力问题
5. 用户原话：「不要顾虑写 反正我自己看」/「我要放在本地站点里看」——本机 Hugo 服务，**不 git commit、不 push**。 如果后来用户说「git add」/「发布」而**没有说「本地」**，那是语义变化，要重新对齐
6. **聊天里的名人辩论内容**虽然有教育意义，但在中国大陆/香港语境下**政治敏感**：用户自己也知道（「反正我自己看」）。 在 angelife 项目上，**默认只写本机草稿**，除非用户明确说「发布」「上线」「git push」

## 极少见地、有「什么真的对了」

- 最后 4 篇文章（T02-T05）连续 4 次都成功：3,050 / 3,593 / 4,097 / 4,726 bytes 实测。 为什么？因为到那时 `[WRITE TASK]` 仪式已经机械化，每篇都短到 stream 不再截断（≈1500-2000 字）
- 最终 5 文件（T01-T03-T02-T04-T05）的输出组成一个**完整系列**，即使经过混乱。 有些文件从零重写过（T02），但**终态不错**

## 一句话教训

**「我准备写」不等于「我已经写」。 LLM 聊天输出是意图声明，不是操作记录。 工具返回结果是唯一可信的真相。**
