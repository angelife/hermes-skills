# 扩散收敛法 — 与 AI 共建的结构化提问框架

## 来源

本文提炼自公众号文章《分享一个「扩散收敛法.skill」》，结合在 Hermes Agent 架构下的实践经验。

## 核心理念

**先扩散，后收敛。** 面对复杂问题时，AI Agent 的思考方式应该分为两步：

### 扩散 (Diverge) — 铺开地图

让 AI 帮你展开所有可能性：
- 你不知道的、可能相关的因素
- 那些看似不起眼但会改写整个设计的"暗礁"
- 多个备选方案及其优劣

### 收敛 (Converge) — 锁死决策

每聊清楚一个分叉，就锁定一个决策：
- 决策点列出所有分叉（用表格对比）
- AI 给出最优推荐并标注置信度
- 每轮 AI 只问一个最关键的问题
- 答完后继续下一轮扩散

## 适用场景

- 架构设计决策
- 技术方案评审
- Bug 根因排查（复杂、多因素）
- 项目起步时的方向选择

## 与 Hermes 的结合

### 在 ask-chatgpt 流程中使用

1. **扩散阶段**：把问题背景、已尝试的方案、具体报错推给 ChatGPT，要求它铺开所有可能的原因和方案。每项标注置信度。
2. **收敛阶段**：逐个锁死决策，每一步只确认一个关键选择。
3. **收口**：所有决策锁完后，让 ChatGPT 输出一份完整的实施方案文档。

### 在 web_search / 内部推理中使用

同样适用于通过 web_search 做技术调研：先广泛收集信息，再逐个缩小范围锁定最佳方案。

## 启发来源

源自设计领域的双钻模型（Double Diamond）和心理学中的发散/收敛思维（Guilford）。本质上是将人类专家多年的思维框架适配到 AI 协作场景。
